# 授课端前端知识库 - 深度分析

## 1. 项目概述

授课端前端项目是一个基于Vue 3和Pinia的教育直播平台，提供丰富的教学互动功能，包括实时互动、课件管理、学情分析、白板交互等。本文档将重点分析核心的Pinia状态管理系统，以`interact.ts`模块为例进行深度解析。

## 2. 核心Store模块深度分析

### 2.1 interact.ts 模块详解

#### 2.1.1 文件信息与定位

- **文件路径**：`/Users/zouyu/zouyu/授课端/client_pc_main_teacher/src/stores/modules/interact.ts`
- **store ID**：`interact`
- **主要功能**：管理教学互动相关的状态和方法，是整个互动教学系统的核心状态容器

#### 2.1.3 内部依赖关系分析

- **database依赖**：
  - `database.courseware.curPage`：获取当前课件页面ID，用于确定当前页的任务列表
  - 通过tasks计算属性，与database紧密结合，实现任务与课件页面的关联

- **broadcast依赖**：
  - `sendMsgSyncStore('courseWareLoading', data.show)`：当加载状态改变时，通过广播机制同步到其他组件
  - 实现了跨组件的状态同步，特别是loading状态的全局协调

- **Queue依赖**：
  - 使用`Queue`类管理互动任务队列，实现互动任务的有序处理
  - 支持复杂场景下的互动任务调度

#### 2.1.4 状态更新触发的副作用

- **loading状态变化**：通过`sendMsgSyncStore`广播通知其他组件，可能触发加载动画显示/隐藏
- **当前互动变化**：影响多个计算属性（如`isInteractionOnGoing`、`curInteractType`等），进而影响UI展示
- **计时器状态**：`timer`和`delayStartTimer`的变化会影响互动的计时和延迟执行逻辑

#### 2.1.5 关键逻辑分析

1. **当前任务获取逻辑**：

### 2.2 类型定义详细分析

#### 2.2.1 核心接口与类型

| 类型/接口名 | 说明 | 主要属性 |
|------------|------|---------|
| InteractDataType | 互动数据核心类型 | id, taskType, status, interactionId, taskExt等 |
| TaskMap | 任务映射类型 | `{[key: string]: InteractDataType[]}` |
| InteractType | 互动类型枚举 | YES_NO, COIN_BANK, PRAISE_BOARD, PK_GAME等 |
| TaskType | 任务类型枚举 | START_GAME, PK_RANK, PRAISE_BOARD等 |
| InteractStatus | 互动状态枚举 | UNSTART, ONGOING, COMPLETED, SKIPED等 |
| Deduction | 任务预置类型枚举 | PREPARE(预置), NOPREPARE(非预置) |

#### 2.2.2 类型关系分析

- **InteractDataType与TaskType关系**：
  - `InteractDataType.taskType`属性使用`InteractType`或`TaskType`枚举值
  - TaskType是对InteractType的细化，如START_GAME是InteractType.GAME_DEMO的子类型

- **TaskMap与InteractDataType关系**：
  - TaskMap是页面ID到InteractDataType数组的映射
  - 一个页面可以有多个互动任务

- **InteractStatus与互动生命周期**：
  - 互动经历UNSTART → ONGOING → COMPLETED/SKIPED的状态流转
  - 部分特殊场景下会出现AGAIN(再次发起)和PREVIEW(预览)状态

### 3.2 常见状态流转示例

1. **互动发起流程**：
   - 初始状态：curInteract = null, isInteractionOnGoing = false
   - 发起互动：设置curInteract，状态为InteractStatus.UNSTART
   - 开始互动：更新curInteract.status为InteractStatus.ONGOING，isInteractionOnGoing变为true
   - 结束互动：更新curInteract.status为InteractStatus.COMPLETED，isInteractionOnGoing变为false

2. **页面切换时的任务更新**：
   - 页面切换 → database.courseware.curPage变化
   - tasks计算属性重新计算，返回新页面的任务列表
   - 可能触发相关UI组件的更新

### 3.3 Store间协作方式

1. **与courseware store协作**：
   - interact store依赖courseware的当前页面状态
   - courseware组件通过interact store获取和更新互动状态

2. **与takePhoto store协作**：
   - 在拍照合影功能中，takePhoto store与interact store配合使用
   - interact store管理互动状态，takePhoto store管理具体的拍照配置和数据

3. **与tab store协作**：
   - tab状态变化可能触发互动UI的显示/隐藏

## 5. 扩展知识 - 互动类型系统详解

### 5.1 互动类型层次结构

1. **顶级互动类型(InteractType)**：定义了主要的互动类别，如PK_GAME, SINGLE_GAME, COIN_BANK等
2. **任务类型(TaskType)**：对顶级类型的细化，如START_GAME是游戏类型的起始阶段，PK_RANK是PK游戏的排行榜阶段
3. **任务形式(TaskForm)**：区分任务是提示型(prompt)还是发起型(initiated)

### 5.2 互动数据模型

- **基础数据(InteractDataType)**：包含所有互动共有的属性
- **扩展数据(taskExt)**：通过taskExt字段实现不同互动类型的特有数据扩展
- **任务映射(TaskMap)**：通过页面ID组织任务，实现任务与课件页面的关联
