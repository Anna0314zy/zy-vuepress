## 动画数据配置完整解析

### 一、核心数据结构（类型定义在 `@icw/typing`）

**数据层级：Page → Animations（步骤组） → AnimationItem（单个动作）**

```
Page.animation: Animations[]          // 默认动画组
Page.animationGroups: [               // 自定义动画组（可多个）
  { id, title, animation: Animations[] }
]
```

[`Animations`](file:///Users/zouyu/Desktop/huohua/icw-creator/packages/typing/src/widgets/animations/index.ts) — 一个"步骤节点"：
```ts
{
  order: number;          // 步骤序号（第几步）
  key: string;            // 唯一 key
  animations: AnimationItem[];  // 这一步里的所有动作
}
```

[`AnimationItem`](file:///Users/zouyu/Desktop/huohua/icw-creator/packages/typing/src/widgets/animations/index.ts) — 一个具体动作：
```ts
{
  key: string;
  type: 'fade-in' | 'fly-out' | 'move' | 'pathMove' | 'rotation' | 'scale' | 'spine' | 'audio' | ...
  delay: number;          // 延迟时间
  start: 'click' | 'auto' | 'sequence' | 'spawn'  // 触发方式
  widgetId: string;       // 作用在哪个 widget 上
  position: [order, index];  // 二维坐标定位 [步骤序号, 步骤内下标]
  props?: {
    duration?: number;    // 持续时间
    direction?: string;   // 方向（fly-in/out 用）
    audioAsset?: string;  // 音频资源
    spineName?: string;   // Spine 动画名
    [key: string]: any;
  }
}
```

### 二、动画类型分类（在 [`data.ts`](file:///Users/zouyu/Desktop/huohua/icw-creator/apps/editor/src/pages/Main/PropertiesPanel/ElementProps/components/AnimationsPane/data.ts) 中配置 UI）

| 分类 | 动画类型 |
|---|---|
| 进入/退出 | `fly-in`, `fly-out`, `bounce-in`, `bounce-out` |
| 强调 | `move`, `pathMove`, `shake`, `heartbeat`, `scale`, `flip`, `rotation` |
| 透明度 | `fade-in`, `fade-out`, `blink` |
| 显隐 | `show`, `hide` |
| 特殊 | `spine`（Cocos Spine动画）、`audio`/`stopAudio`、`switchImage`、`micro` |

每种类型在 `data.ts` 里声明其可配置的 `props` 表单项（如 number 输入框、select 下拉框），编辑器通过 `getFormItenconfig(type)` 动态渲染属性面板。

### 三、互斥规则（`rule` 配置）

在 [`data.ts`](file:///Users/zouyu/Desktop/huohua/icw-creator/apps/editor/src/pages/Main/PropertiesPanel/ElementProps/components/AnimationsPane/data.ts) 中有 `rule` 对象，规定了同一元素不能同时存在的动画类型：

```ts
'fly-in': ['move', 'shake', 'fly-out', 'fly-in'],  // fly-in 与这些互斥
'fade-in': ['blink', 'fade-out', 'fade-in'],
```

### 四、动画组机制

```
系统默认组 (editingAnimation === 'system')
   ↓ page.animation: Animations[]

自定义动画组 (editingAnimation === group.id)
   ↓ page.animationGroups[i].animation: Animations[]
```

动画组可以被 **Interaction（交互）** 触发：点击某个 widget → `playAnimation` → `actionValue = group.id`

### 五、数据流：编辑 → 存储 → 播放

```
用户在面板操作
   ↓
PropertiesPanel/AnimationsPane（React UI）
   ↓
tsyringe Command（AddAnimation / UpdateAnimationItem...）
   ↓
effects/animations.ts（updateAnimationListItemProps）
   ↓
store.dispatch.main.updateAnimationList({ pageId, group, animations })
   ↓ 存入 Redux state: main.pages[n].data.animation
   ↓
Canvas/EditCanvas/animation.ts（Fabric.js 绘制路径预览线）
   ↓ 运行时
extensions/widgets/assets/runtime/animation.ts（Cocos cc.Tween 执行动画）
```

## Editor 项目架构详解

### 一、整体布局结构

```
EditorLayout (Main/index.tsx)
├── SubHeader          顶部工具栏（保存、发布、预览、导入...）
├── SliderMenu         左侧面板（幻灯片结构树 + 图层）
├── Content（中间区域）
│   ├── Canvas         画布容器（缩放/平移控制）
│   │   ├── Player     Cocos iframe 预览层（只读）
│   │   └── EditCanvas Fabric.js 编辑层（可交互）
│   └── LogicCanvas    逻辑编辑器（ReactFlow 节点图）
├── PropertiesPanel    右侧属性面板
└── PrimaryModal       全局弹窗
```

---

### 二、双画布叠加机制（核心设计）

EditCanvas 是**两层叠加**的：

```
<div id="dom-canvas-wrapper">          ← transform3d 控制位移/缩放
  <canvas ref={CanvasDom} />           ← Fabric.js Canvas（底层，处理选中/拖拽/变换）
  <div id="dom-canvas-wrapper">        ← DOM 浮层（动画序号标注、答案配置、文本编辑器...）
</div>
<div id="responsive">
  <Player />                           ← Cocos iframe（渲染实际 widget 画面）
</div>
```

- **Fabric.js 层**：不渲染 widget 视觉，只渲染透明的占位框（用于选中、拖拽、缩放、对齐辅助线）
- **Cocos iframe 层**：实际渲染 widget 视觉，通过 `post-me` 与编辑器通信
- **DOM 浮层**：React 渲染动画序号、答题配置、文本编辑器气泡等覆盖元素

[`CanvasEditor.ts`](file:///Users/zouyu/Desktop/huohua/icw-creator/apps/editor/src/pages/Main/Canvas/EditCanvas/CanvasEditor.ts) 是编辑器的核心，直接持有 Fabric.js 实例，并组合多个功能模块：

class EditorCanvas {

  animationModule: AnimationModule  // 动画路径绘制（贝塞尔曲线）
  spectModule: SpectModule          // 对齐/吸附规格检查

[`EditorBridge`](file:///Users/zouyu/Desktop/huohua/icw-creator/apps/editor/src/pages/Main/Canvas/EditCanvas/editorBridge.ts) 是一个全局单例，作为 React 组件与 `CanvasEditor` 之间的桥梁：

```
React组件 → editorBridge.setWidgets() → CanvasEditor.setObjects()
React组件 → editorBridge.setSelection() → CanvasEditor.setSelection()
```

### 四、Command 模式（DI 驱动）

所有用户操作都封装为 Command，通过 **tsyringe** IoC 容器解析：

```
用户操作
   ↓
command.resolve(delay(() => AddWidgets))   // 懒加载解析依赖
   ↓
class AddWidgets extends Base {
  constructor(@inject History) {}          // 自动注入 History 服务
  async run() {
    store.dispatch.main.xxx()              // 修改 Redux state
    this.updateHistory(pageId)             // 写入 undo 历史
  }
}
```

Command 分类：
- `command/page/`：widget 增删改（addWidgets、deleteWidgets、updateWidget...）
- `command/animation/`：动画增删改
- `command/project/`：项目初始化、加载
- `services/`：History（撤销栈）、HotKey（快捷键）、AssetManager（资源管理）

---

### 五、撤销/重做机制

[`History`](file:///Users/zouyu/Desktop/huohua/icw-creator/apps/editor/src/commands/services/history.ts) 是一个 `@singleton()` 服务，按 `pageId` 分别维护各页的历史栈：

```ts
_history: { [pageId]: Page[] }   // 每页独立历史，最多50条
_index:   { [pageId]: number }   // 当前指针

push(pageId, page)  → cloneDeep 快照存入
undo(pageId)        → 指针-1，updateCanvasView 重绘
redo(pageId)        → 指针+1，updateCanvasView 重绘
```

注意：undo/redo **不回滚** `animation`、`answers`、`interactions`（这些单独处理），只回滚 widget 位置/属性。

### 六、数据持久化流程

```
用户操作
   ↓
Rematch dispatch（action.type 以 "main/" 开头）
   ↓
realTimeSave 中间件拦截
   ↓
savePageJson(icwCode, pageId)       // debounce 1000ms
   ↓
updatePageJson()
  ├── clearPageJson()               // 清理无效资源引用
  ├── validate(state)               // @icw/schema JSON Schema 校验
  ├── gzipStr(json)                 // gzip 压缩
  ├── pageSave(API)                 // 发送到服务端
  ├── cachePageJson(localforage)    // 写入 IndexedDB 本地缓存
  └── BroadcastChannel.postMessage  // 通知其他 tab（预览窗口）同步
```

### 七、页面类型与右侧面板切换

`global.editMode` 和 `global.currentPage.type` 共同决定右侧面板展示什么：

```
editMode === 'logic'     → LogicBlocks（逻辑节点配置面板）
pageType === 'game'      → GameProps（Cocos 游戏专属属性）
其他                     → ElementProps（普通 widget 属性 + 动画 + 答案）
```
