# Player-v2 授课端 & 学生端同步机制分析

### 总结一句话

Player-v2 存在**两套独立的同步机制**，分别对应两种课堂形态：

---

## 一、普通课堂（AS / SE 平台）：基于 `RPCBridge` + Native 客户端消息总线

### 架构模型

```
[授课端 App (Native)]
      ↕ RPCBridge / IPC
[Player-v2 WebView]
      ↕ rpcClient.emit / on
[学生端 App (Native)]
      ↕ RPCBridge / IPC
[Player-v2 WebView (学生)]
```

Player 本身**不直接连学生端**，它只负责和宿主 Native App 通信。**翻页同步的通路在 Native 层**，Native 负责把老师的操作广播给所有学生的 Native，再由学生 Native 驱动学生的 Player。

### 关键消息流

**老师翻页时（教师端 Player）：**
```typescript
// events.ts - playerOnChangePage 触发后
MessageService.rpcClient.emit('change', {
  page,          // 当前页码
  uid,           // 老师的用户ID
  code,          // slide code
  state: {}      // 当前页交互状态
})
```
→ `rpcClient.emit('change', ...)` 把事件上报给 Native App

**学生端 Player 接收：**
```typescript
// events.ts - bindRpcEvents
rpcClient.on('change', changeEventHandler)
```
→ Native 把翻页事件推给学生 Player，`changeEventHandler` 处理翻页逻辑

**学生端翻页触发点：**
```typescript
// events.ts - changeEventHandler
export function changeEventHandler(data: ChangePagePayload) {
  let { page, code, state } = data
  // 同页只更新state，不同页才翻页
  if (page === -1 || (MessageService.currentPage === page && data.state)) {
    player.changeState(data)     // 只更新交互状态
  } else {
    messageService.debounceChangePage({ ...data, traceID }) // 翻页
  }
}
```

### 同步的数据结构

```typescript
// change 事件 payload
{
  page: number,       // 页码
  code: string,       // slide 的 materialCode
  uid: string,        // 发起者 uid
  state: any,         // 该页的交互状态（答题进度、步骤等）
  cursor?: [x, y, type]  // 光标位置（百分比坐标）
}
```

### 光标同步

```typescript
// events.ts - bindCursorEvents
CursorService.changeCursor = (e: MouseEvent) => {
  const point = [
    e.clientX / window.innerWidth,
    e.clientY / window.innerHeight,
    cursorService.cursorType
  ]
  CursorService.rpcClient.emit('updateCursorPosition', { cursor: point })
}
```
老师鼠标移动 → 以**屏幕百分比坐标**上报 Native → 广播给学生 → 学生 Player 渲染光标

## 二、ULTRA 平台（大班课/小班课）：基于 `$crsdk` MasterStore 发布订阅

这是一套更完整的实时同步方案，通过 `@classpod/classroom-plugin-jssdk` 提供的 **MasterStore** 实现。

### 架构模型

```
[主讲/助教 Player]
   dispatchMasterStoreChange(endpoint, state)
          ↓ CRSDK (RTM 实时消息)
[学生端 Player × N]
   handleStoreChange(endpoint, state)
          ↓
   changeEventHandler(state)  // 触发翻页/状态同步
```

### Channel（频道）设计

每个频道（endpoint）的 URL 格式：
```
/plugin/pcp/{uuid}/{masterId}/{pageCode}
```

共有两类频道：

| 频道类型 | endpoint 格式 | 用途 |
|---|---|---|
| **课堂级** | `/plugin/pcp/{roomCode}/{masterId}/{uuid}` | 翻页同步（page 级别） |
| **页面级** | `/plugin/pcp/{uuid}/{masterId}/{slideCode}` | 页面内交互状态同步（state 级别）|

### 初始化时的订阅逻辑

```typescript
// ultraClass/message.ts - createChannelsForIcw
if (大班模式) {
  if (主讲) → createMasterStore(课堂endpoint)   // 我来发
  else       → subscribeStore(课堂endpoint)      // 我来收
} else if (小班模式) {
  if (助教) → createMasterStore(小班endpoint)
  else      → subscribeStore(小班endpoint)
}
// 同时对每一个 slide 的 pageCode 都创建/订阅一个独立 channel
for (const slide of slides) {
  await creator(getEndpoint(uuid, masterId, slide.code))
}
```

### 翻页同步发送（主讲端）

```typescript
// events.ts - bindPlayerSDKEvents 内
player.emitter.on('changePage', page => {
  const config = getConfig()
  dispatchPluginState({ page }, config)      // 通知课堂级频道：翻到第 X 页
})
```

```typescript
// ultraClass/message.ts - dispatchPluginState
dispatchMasterStoreChange({
  state: { masterId, page },
  endpoint: PluginEndpoint + roomCode + '/' + masterId + '/' + uuid,
})
```

### 页面内状态同步（交互答题等）

```typescript
// events.ts - player.emitter.on('changeState')
player.emitter.on('changeState', state => {
  const payload = { state, page, uid, code, action: 'change' }
  dispatchPageState(payload, config) // 发到页面级频道
})
```

### 学生端接收

```typescript
// ultraClass/message.ts - handleStoreChange
switch (endpoint) {
  case 课堂级endpoint:
    changeEventHandler(state)   // 翻页
    break
  case 当前页面endpoint:
    changeEventHandler(state)   // 更新页面内交互状态
    break
}
```

### 进入课堂时的状态追赶（Late-join）

```typescript
// events.ts - loaded 事件
const initState = await getMasterStoreState(config, config.slides)
// 读取 MasterStore 的最新快照，直接跳到老师当前页+当前状态
await changeEventHandler({ ...initState, traceID: payload.traceID })
```
学生晚进入时，直接拉取 MasterStore 的最新快照，**不需要等老师翻页**就能同步到当前位置。

## 三、两套机制对比

| 维度 | 普通课堂（RPCBridge）| ULTRA 大班课（CRSDK MasterStore）|
|---|---|---|
| **通信通路** | Native App 消息总线 | 实时消息服务（RTM）|
| **Player 角色** | 被动接收 Native 指令 | 主动 publish/subscribe |
| **同步粒度** | 页码 + state | 页码（课堂级）+ state（页面级）|
| **Late-join** | 由 Native 处理 | `getMasterStoreState` 读快照 |
| **光标同步** | emit `updateCursorPosition` | `dispatchPluginState({ cursor })` |
| **学生禁用互动** | `changeEnable(false)` | 检测 `masterId !== uid` 后禁用 |
| **场景识别** | UA 中含 `teacher` 关键字 | `CRSDK.getUserRole()` |

## 四、场景识别逻辑

```typescript
// message.ts - getScene
getScene(classroom?: string) {
  if (classroom) {
    MessageService.config.scene = classroom === 'teacher' ? 'teacherClass' : 'studentClass'
  }
  if (MessageService.config.scene === 'default' && !this.isWeb) {
    const teacher = navigator.userAgent.includes('teacher')  // UA 判断
    MessageService.config.scene = teacher ? 'teacherClass' : 'studentClass'
  }
}
```

普通客户端通过 **User-Agent 中是否含 `teacher`** 来区分角色，ULTRA 则通过 `CRSDK.getUserRole()` 精确获取。
