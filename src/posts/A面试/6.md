 
# react-moveable
---

## 1. 如何知道元素在哪里

用浏览器原生的 `getBoundingClientRect()` 获取目标元素的位置和尺寸，然后在元素周围**绝对定位**渲染出一圈控制点（8个方向的缩放点 + 旋转点）。

---

## 2. 控制点怎么响应拖拽

监听控制点上的 `pointerdown / pointermove / pointerup` 事件，拖动时实时计算鼠标偏移量，转换成对应的 `transform` / `width` / `height` 数值，通过回调函数（`onDrag` / `onResize` / `onRotate`）把结果给你，**它自己不修改任何数据，只是告诉你"变了多少"**。

---

## 3. 控制点位置如何跟着元素走

用 `CSS transform` 同步控制点容器的位置，配合 `ResizeObserver` 和 `MutationObserver` 监听元素变化，自动调用 `updateRect()` 重新计算控制点位置。

---

## 用一句话总结

> react-moveable 本质上是一个"**贴在元素上的控制器 UI**"，它只负责计算变换数值和显示控制点，真正的数据更新由你在回调里完成。

---

这也是为什么这个项目里在 `onDrag` 回调中要**同时更新两个地方**：
- `target.style.transform` → 更新 DOM 视觉（立刻生效，用户看到流畅动画）
- `TreeNode.props.style` → 更新引擎数据（保证状态正确）

## 对接的核心问题：谁选中了谁

引擎有自己的选中状态（`operation.selection.selectedNodes`），Moveable 需要的是 **DOM 元素**。所以对接的核心就是：

**引擎选中节点 → 找到对应 DOM → 传给 Moveable**

```
用户点击画布元素
  → 引擎触发 SelectNodeEvent
  → 拿到被选中的 TreeNode.id
  → viewportElement.querySelector(`[data-designer-node-id="${id}"]`)  找到 DOM
  → setTargets([dom元素])  传给 Moveable
  → Moveable 渲染控制点
```

---

## 反向：Moveable 变换 → 引擎数据更新

```
用户拖动控制点
  → onDrag 回调触发
  → target.dataset.designerNodeId  从 DOM 上拿到节点 id
  → tree.findById(id)  找到 TreeNode
  → treeNode.props.style = { transform: newValue }  写入引擎数据
  → MobX 响应式触发组件重渲染
```

---

## 关键的"桥梁"是两个自定义属性

每个画布元素渲染时，引擎会自动在 DOM 上挂两个属性：

| 属性 | 作用 |
|------|------|
| `data-designer-node-id` | 通过 DOM 反查 TreeNode |
| `class="ld_moveable-item"` | 标记"这个元素可以被 Moveable 控制" |

**有了这两个属性，DOM 和引擎数据就双向打通了。**

---

## 一句话总结

> Moveable 和引擎的对接，本质是通过 DOM 上挂的节点 ID 属性做双向映射——引擎选中事件触发时查 DOM 传给 Moveable，Moveable 回调时从 DOM 读 ID 再找回引擎节点更新数据。
