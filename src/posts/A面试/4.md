## 1. 先定义所有互动类型
```ts
// 互动类型
type InteractionType = 'singleChoice' | 'handUp' | 'vote'

// 互动全局状态
type InteractionStatus = 'idle' | 'running' | 'waitingQueue'
```

## 2. 抽象基类（所有互动共用逻辑）
倒计时、广播、排队、互斥、结束、重置**全部写这里**
子类不用重复写
```ts
abstract class BaseInteraction {
  public type: InteractionType
  public status: InteractionStatus = 'idle'
  public duration: number // 统一倒计时时长

  constructor(type: InteractionType, duration = 30) {
    this.type = type
    this.duration = duration
  }

  // ========== 所有互动完全通用逻辑 ==========
  // 发起互动
  public start() {
    // 全局加锁，禁止同时多个互动
    InteractionManager.lock()
    this.status = 'running'
    this.broadcastToAllStudent() // 全员同步
    this.startCountDown() // 统一倒计时
    this.onStartRender() // 差异化渲染钩子
  }

  // 全员广播消息（老师→所有学生）
  protected broadcastToAllStudent() {
    socket.emit('interactive_start', {
      type: this.type,
      status: this.status
    })
  }

  // 统一倒计时
  private startCountDown() {
    setTimeout(() => {
      this.timeOver()
    }, this.duration * 1000)
  }

  // 时间结束
  public timeOver() {
    this.collectResult()
    this.showResult()
    this.close()
  }

  // 关闭释放
  public close() {
    this.status = 'idle'
    InteractionManager.unlock() // 解锁，下一个互动可以进队列
    InteractionManager.nextQueue() // 自动执行排队里下一个
  }

  // ========== 子类必须自己实现 差异化逻辑 ==========
  protected abstract onStartRender(): void       // 不一样弹窗样式
  protected abstract handleStudentAnswer(res: any): void // 不一样答题逻辑
  protected abstract collectResult(): void      // 不一样统计
  protected abstract showResult(): void         // 不一样结果展示
}
```

## 4. 全局单例互动管理器（管顺序、排队、互斥）
**课堂整个App唯一**
课件、工具栏、侧边栏、右键菜单全都找它发起
```ts
class InteractionManager {
  static instance: InteractionManager
  private isLock = false // 全局互斥锁
  private queue: BaseInteraction[] = [] // 互动顺序队列

  // 单例
  static getInstance() {
    if (!this.instance) this.instance = new InteractionManager()
    return this.instance
  }

  // 外部统一调用入口（任何地方都用这个）
  startInteraction(type: InteractionType, params: any) {
    const interaction = this.createInstance(type, params)

    if (this.isLock) {
      // 正在进行中 → 加入队列，按顺序等待
      this.queue.push(interaction)
      return
    }

    // 空闲 → 直接执行
    interaction.start()
  }

  // 根据类型创建对应互动实例
  private createInstance(type: InteractionType, params: any) {
    switch (type) {
      case 'singleChoice': return new SingleChoiceInteraction(params)
      case 'handUp': return new HandUpInteraction(params)
      case 'vote': return new VoteInteraction(params)
    }
  }

  // 上锁
  lock() { this.isLock = true }
  // 解锁
  unlock() { this.isLock = false }

  // 队列自动执行下一个
  nextQueue() {
    if (this.queue.length === 0) return
    const next = this.queue.shift()!
    next.start()
  }
}
```

全部一行代码发起，不用关心顺序、不用关心互斥

## 6. 运行逻辑一眼看懂
1. 老师点发起 → 调用管理器
2. 管理器检查**有没有正在跑的互动**
   - 有：放进队列，排队按顺序执行
   - 没有：直接启动
3. 父类统一走：广播 → 倒计时 → 结束
4. 子类只负责：长什么样、怎么收答案、怎么出结果
5. 互动结束 → 自动解锁 → 取出队列下一个自动执行

## 7. 这套架构优势完全贴合你的需求
1. **相同逻辑复用**：广播、倒计时、排队、互斥全通用
2. **不同逻辑隔离**：每种互动自己改自己渲染、统计
3. **严格顺序控制**：串行队列，绝不会同时弹出多个互动
4. **全局多处调用**：所有组件只调管理器，不互相耦合
5. 新增互动**极快**：继承基类，只写4个方法就行
