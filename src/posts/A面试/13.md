# Node.js MVC + RESTful 接口 面试大白话+标准答案
## 1、MVC 是什么（Node 后端三层架构）
- **Model 模型**：数据库操作、数据处理、校验、实体
- **View 视图**：后端一般不用页面，只返回 JSON 数据
- **Controller 控制器**：接收请求 → 调用Model处理数据 → 返回结果

分层逻辑：
路由 → Controller（业务逻辑）→ Model（数据库）→ 返回REST格式JSON

---

## 2、RESTful API 规范（必背）
用**HTTP请求方式**代表操作动作，不用接口名写add/delete/update
1. `GET` 查询数据
2. `POST` 新增数据
3. `PUT` 全量修改
4. `PATCH` 局部修改
5. `DELETE` 删除数据

URL只用**名词复数**
`/users` `/goods` 而不是 `/getUser` `/addUser`

返回统一JSON格式：
状态码200/201/400/401/403/404/500
code、data、msg 结构

---

## 3、Node.js MVC 实际项目结构（简历写法）
1. 路由层 routes：分发接口地址
2. 控制层 controller：处理业务逻辑、参数校验
3. 模型层 model：Sequelize/Mongoose 操作MySQL/MongoDB
4. 中间件 middleware：鉴权、跨域、异常捕获、日志
5. 配置层 config：数据库、环境变量
