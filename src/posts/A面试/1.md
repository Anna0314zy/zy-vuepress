# 一、micro-app 
- 主子应用**资源预加载 / 懒加载**，减少首屏阻塞
- 公共依赖**抽离共享**（Vue、VueRouter、axios 等），避免重复加载
- 子应用**按需注入**，非激活状态**卸载 JS/CSS**，降低内存占用
- 开启**沙箱隔离优化**，减少样式冲突与冗余渲染
- 主子应用通信**防抖节流**，避免频繁触发渲染
- 静态资源**CDN 加速**，子应用独立打包并行加载
- 子应用打包**external 外部化**，减小构建体积

# 二、部署方案（简历专业表述）
- 主子应用**独立域名/独立路径**部署，Nginx 反向代理转发
- 子应用**独立构建、独立部署**，支持增量更新不影响主应用
- 配置跨域、资源路径 publicPath，实现主子应用资源正常加载
- 支持**多环境部署**（开发/测试/生产），环境变量隔离
- 结合 CI/CD 实现微前端应用自动化构建与发布

## 渲染优化
具备基于 micro-app 的微前端落地经验，负责 Vue 项目拆分解耦、应用间通信、公共依赖抽离、沙箱隔离与懒加载优化，以及主子应用独立部署与 Nginx 代理配置。
长列表优化：
数据量大时页面卡顿，用 虚拟滚动（vue-virtual-scroller） 只渲染可视区域，减少 DOM 数量。
渲染优化：
减少不必要渲染，用 computed 缓存、v-once、拆分组件、keep-alive 缓存页面，避免重复渲染。
通信优化：
组件多层嵌套时，用 mitt /provide-inject/ 全局状态 替代频繁 props 传递，避免链式更新导致卡顿。
打包优化：
路由懒加载、代码分割、第三方库 CDN、图片压缩，减小首屏体积。
