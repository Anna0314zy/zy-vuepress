# 大纲树
---

## 大纲树层级拖拽实现原理

这套拖拽**不依赖任何第三方拖拽库**，完全是引擎自己实现的，核心是 **MoveHelper + 事件系统**。

---

### 核心角色

| 角色 | 职责 |
|------|------|
| `useDragDropEffect.ts` | 监听鼠标事件，驱动整个拖拽流程 |
| `MoveHelper` | 拖拽过程中的状态机（谁在拖、拖到哪、插入位置） |
| `ClosestPosition` | 枚举：Before / After / Inner / Upper / Under 等 |
| `OutlineNode.tsx` | 大纲树节点 UI，响应拖拽状态显示高亮 |

---

### 拖拽流程

**1. DragStart — 记录被拖的节点**

鼠标按下时，从 DOM 上读 `data-designer-outline-node-id` 属性找到对应的 `TreeNode`，调用 `moveHelper.dragStart({ dragNodes })`。

**2. DragMove — 实时计算插入位置**

鼠标移动时，`moveHelper.dragMove()` 做两件事：
- `calcClosestNode()`：找当前鼠标最近的节点（`closestNode`）
- `calcClosestPosition()`：根据鼠标在 closestNode 矩形区域的位置，判断插入方向：
  - 鼠标在节点**上半区** → `Upper`（插前面）
  - 鼠标在节点**下半区** → `Under`（插后面）
  - 鼠标在节点**内部中心** → `Inner`（变成子节点）

OutlineNode 监听 `DragMoveEvent`，检测自己是不是 `closestNode`，是就加 `droppable` 高亮 class，悬停 600ms 自动展开。

**3. DragStop — 执行插入**

松开鼠标，根据 `closestDirection` 调用 TreeNode 对应方法：

```
After / Under   → closestNode.insertAfter(dragNodes)
Before / Upper  → closestNode.insertBefore(dragNodes)
Inner           → closestNode.append(dragNodes)
InnerBefore     → closestNode.prepend(dragNodes)
```

TreeNode 的 `insertAfter` / `append` 等方法直接操作 `children` 数组，MobX 感知变化后自动重渲染大纲树和画布。

---

### 一个关键保护逻辑

```ts
if (isMoveStart === 'node') return moveHelper.dragEnd()
```

从**画布上**拖动元素时，直接阻止修改层级 —— 画布拖拽只改坐标，不改 TreeNode 的父子关系，防止误操作。

---

**一句话总结**：大纲树拖拽是引擎自己的事件驱动系统实现的，鼠标每次移动都实时计算"最近节点"和"插入方向"，松手时直接调 TreeNode 的 insert/append 方法改树结构，MobX 自动触发视图更新。
