# 动画
---

## 动画系统整体结构

**三层组成**：

```
配置层（animate_css.ts）  →  数据层（page.props.animates）  →  执行层（AnimateEngine）
    动画有哪些                  当前页存了哪些动画配置           真正播放动画
```

---

## 第一层：配置层

所有可用的动画都预先定义在 `animate_css.ts` 里，每条配置长这样：

```js
{
  name: "弹跳",        // 面板里显示的名字
  value: "bounceIn",  // 对应 CSS @keyframes 的名字
  type: "entrance",   // 入场 / 出场 / 强调
  directions: [       // 支持哪些方向
    { key: "general", value: "bounceIn" },
    { key: "up",      value: "bounceInUp" },
    { key: "down",    value: "bounceInDown" },
  ]
}
```

选动画时就是从这个配置表里选，**动画面板的下拉菜单是动态从这个配置表生成的**。

---

## 第二层：数据层

每条动画配置选好后，作为一个对象存进 `page.props.animates` 数组里：

```js
// 存在页面 Root 节点的 props 上
page.props.animates = [
  {
    id: "xxx",
    target: "组件id",        // 作用在哪个元素上
    type: "entrance",        // 入场
    name: "bounceIn",        // 具体动画名
    trigger: "click",        // 触发方式：点击 / 自动 / 与上一个同时 / 上一个之后
    triggerSource: "workspace", // 由谁触发：画布 or 另一个组件id
    direction: "general",    // 方向
    duration: 0.5,           // 时长（秒）
    delay: 0,                // 延迟（秒）
  }
]
```

**改任何配置（方向/时长/触发方式）都是直接修改这个数组里的对象，MobX 响应式自动更新面板显示。**

---

## 第三层：执行层

播放时 `AnimateEngine.play()` 做的事很简单，就是把配置翻译成 CSS animation 字符串打到 DOM 上：

```js
// 生成的 CSS 字符串
"0.5s ease 0s 1 normal none running bounceInDown"
//  时长  缓动 延迟          次数                  动画名

el.style.animation = 这个字符串  // 直接挂到元素上播放
```

动画结束后监听 `animationend` 事件，自动清除 animation 样式，恢复原始状态。

---

## 触发方式是怎么实现的

| 触发方式 | 实现原理 |
|---------|---------|
| 单击时 | 点击画布时遍历 animates，找到 trigger=click 的播放 |
| 自动播放 | 进入页面时自动执行 |
| 与上一个同时 | 上一个动画开始时，同时 play 这个 |
| 上一个之后 | 监听上一个 animationend，再 play 这个 |

---

## 一句话总结

> 动画系统的本质是：**预设动画配置表 + 用户选配存进页面数据 + 播放时把数据翻译成 CSS animation 字符串打到 DOM 上**，整个系统没有复杂的动画引擎，核心就是 CSS keyframes + 数据驱动配置面板。
