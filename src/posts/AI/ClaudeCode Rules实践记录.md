# Claude Rules 使用说明

## 1. 存放位置

规则文件通常放在项目根目录：

```
.claude/rules/
```

核心配置文件：

```
CLAUDE.md
```

用于定义 **项目级全局规则**。

### 文件结构

规则文件结构很简单：

```
YAML 元数据
---
Markdown 规则正文
```

示例：

```yaml
---
description: Frontend coding rules
globs:
  - "**/*.ts"
  - "**/*.vue"
priority: 1
---

# Coding Rules

- 使用 TypeScript
- 不允许使用 any
- 组件使用 Composition API
```

说明：

| 部分          | 作用           |
| ----------- | ------------ |
| YAML 元数据    | 指定规则适用文件、优先级 |
| Markdown 正文 | 具体规则内容       |

---

# 2. 常用命令

在 Claude 会话中可以使用以下命令管理规则。

### 查看规则

```
/rules
```

查看当前会话中 **所有生效规则**。

---

### 添加临时规则

```
/add-rule
```

在当前会话 **临时添加一条规则**。

---

### 清除临时规则

```
/clear-rules
```

删除当前会话中 **临时添加的规则**。

---

### 查看会话状态

```
/status
```

查看：

* 当前会话状态
* 当前规则应用情况

---

# 3. 核心行为准则

Claude 在执行任务时应遵循以下原则：

* 优先理解用户需求
* 保持回答简洁清晰
* 优先复用已有代码
* 避免无必要的复杂设计
* 提供可执行方案

---

# 4. 工具使用规则

## 优先使用专用工具

| 操作   | 推荐工具   | 不推荐    |
| ---- | ------ | ------ |
| 读取文件 | `Read` | `cat`  |
| 搜索文件 | `Grep` | `grep` |

---

## 并行执行

如果多个任务 **相互独立**，应同时调用多个工具执行。

例如：

* 同时搜索多个文件
* 同时读取多个配置文件

---

## 风险操作需确认

以下操作必须 **用户确认**：

* 推送代码
* 删除分支
* 修改重要配置
* 删除文件

---

# 5. 会话记忆规则

Claude 支持持久化记忆。

存储目录：

```
~/.claude/projects/
```

### 默认文件

```
MEMORY.md
```

限制：

```
最多 200 行
```

---

### 可以创建专题记忆文件

例如：

```
patterns.md
debugging.md
architecture.md
```

用途示例：

| 文件              | 用途     |
| --------------- | ------ |
| patterns.md     | 常用代码模式 |
| debugging.md    | 常见问题排查 |
| architecture.md | 项目架构   |

---

# 6. 禁止行为

Claude 必须避免以下行为：

* ❌ 不猜测或生成 URL
* ❌ 不重复已被拒绝的工具调用
* ❌ 不为简单任务进行过度设计

---

# 7. Rules 文件大小限制

建议控制规则文件规模：

* 单个 Rules 文件 **尽量简洁**
* 避免过多无关规则
* 按模块拆分规则文件

推荐结构：

```
.claude
 ├─ rules
 │   ├─ frontend.md
 │   ├─ backend.md
 │   ├─ git.md
 │   └─ testing.md
```
